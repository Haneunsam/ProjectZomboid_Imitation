# Project Zomboid Imitation - AI Collaboration Guidelines

이 파일은 프로젝트 좀보이드 모작 프로젝트를 진행할 때 AI가 지켜야 할 규칙과 프로젝트 배경을 설명합니다. 새로운 대화를 시작할 때 이 내용을 읽어주세요.

## 1. 프로젝트 개요
- 언리얼 엔진 5.3 기반의 탑뷰(아이소메트릭) 좀비 생존 게임 모작.
- 기간: 2주 집중 개발.
- 목표: 좀보이드 특유의 조작감, 인벤토리, 생존 시스템 구현.

## 2. 협업 규칙 (매우 중요)
1. **계획 우선**: 모든 요청에 대해 실행 계획을 먼저 수립하고 사용자에게 공유한다.
2. **단계별 실행**: 승인된 계획에 따라 하나씩 차례대로 진행한다.
3. **로직 및 구조 중심**: 코드를 직접 대량으로 생성하기보다, 시스템의 구조와 로직의 흐름을 설명하는 데 집중한다.
4. **직접 타이핑**: 사용자가 코드를 직접 손으로 칠 것이므로, 가독성 좋은 코드 스니펫과 구조도를 제공한다.
5. **오류 수정**: 사용자가 작성한 코드에 에러가 발생하거나 논리적 결함이 있을 때만 수정을 제안한다.
6. **설정 가이드**: 코드 외에 언리얼 에디터(IMC, BP, Project Settings 등)에서 설정해야 할 세부 사항을 상세히 알려준다.

## 3. 핵심 조작 로직
- 이동: 카메라 기준이 아닌, 월드 대각선 방향(W = X+Y+) 이동 방식을 사용한다.
- 회전: 마우스 커서 방향을 실시간으로 추적하여 캐릭터가 회전한다.
